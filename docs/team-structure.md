# チーム構成・役割

## 概要

- **3チーム × 4名** = 計12名
- 各チームが異なるシナリオ（A / B / C）を担当
- 発表会で各チームの成果を共有し、多様な学びを得る

## 役割分担

各チームは以下の4つの役割を分担します。1名1役が基本ですが、状況に応じて柔軟に協力してください。

### BA / PM 役（1名）

- ヒアリング・要件整理・発表の担当
- M365 Copilot を活用して業界リサーチを実施
- 要件定義書・議事録の生成
- 発表会でのプレゼンテーションをリード

### アーキテクト役（1名）

- マルチエージェント構成設計
- 全体アーキテクチャの統括
- Foundry + GitHub Copilot を活用した設計
- 技術的な意思決定のリード

### デベロッパー ① — アプリ実装（1名）

- フロントエンド・API 統合
- テスト・デバッグ担当
- GitHub Copilot を中心に実装

### デベロッパー ② — Azure / Foundry（1名）

- Foundry IQ（Agentic RAG）の実装
- インデックス設計・チューニング
- エージェント間連携の実装

## タイムライン別の動き方

| フェーズ | BA/PM | アーキテクト | Dev① | Dev② |
|----------|-------|------------|-------|-------|
| リサーチ・ヒアリング準備 | M365 Copilot でリサーチ主導 | 技術的な質問を準備 | 質問リスト支援 | 質問リスト支援 |
| 顧客ロールプレイ | メインインタビュアー | 技術質問担当 | 記録係 | 補足質問 |
| 設計 | 要件書ドラフト作成 | アーキテクチャ設計主導 | 実装可能性の検証 | Foundry 構成設計 |
| 実装 | テストデータ準備・進捗管理 | レビュー・技術支援 | アプリ実装 | エージェント実装 |
| 発表準備 | プレゼン構成・ナレーション | デモシナリオ設計 | デモ環境準備 | デモ環境準備 |
