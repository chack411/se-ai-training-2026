## 進め方

各チームは割り当てられたシナリオに沿って、以下の流れで進めます。

### 1. シナリオブリーフィング（15分）

- チーム内で README.md と `data/` フォルダのサンプルデータを確認
- 顧客像・業界背景・課題の概要を把握

### 2. リサーチ（〜1時間）

- M365 Copilot を活用して業界背景を深掘り
- 顧客の課題に関連する技術トレンドを調査
- ヒアリングで聞くべき質問リストを作成

### 3. 顧客ロールプレイ（20分 / チーム）

- トレーナーが顧客役を務めます
- **聞かれたことだけ回答する** 形式 — 積極的にヒアリングしてください
- 時間内にできるだけ多くの情報を引き出すことが重要

### 4. 要件整理・設計（〜45分）

- ヒアリング結果を要件書にまとめる（M365 Copilot 活用）
- Microsoft Foundry を使ったマルチエージェント構成を設計
- トレーナー（顧客役）に設計を説明し、方向性を合意

### 5. 実装（〜5.5時間）

- GitHub Copilot を活用した Vibe Coding
- Microsoft Foundry でマルチエージェントを構築
- `data/` フォルダのサンプルデータを活用してプロトタイプを動作させる

### 6. 発表（25分 / チーム）

- デモ 15分 + Q&A 10分
- トレーナーが顧客目線でフィードバック

## ヒアリング時の共通注意事項

- **一方的に提案しない** — まずは顧客の言葉で課題を引き出すことに徹する
- **「なぜ？」を繰り返す** — 表面的な要望の裏にある真のペインを探る
- **全員が発言する** — チーム全員が役割を持ち、多角的に質問する
- **メモを取る** — 後の設計・提案に向けて、具体的な数値・発言を記録する
- **時間配分を意識** — 全ポイントをカバーできるよう計画的に進行する

## サンプルデータについて

各シナリオの `data/` フォルダに、ロールプレイで扱うサンプルデータが格納されています。これらは実装フェーズで RAG のデータソースやエージェントの入力データとして活用してください。

| シナリオ | データ内容 |
|----------|-----------|
| A — 製造業 | 技術マニュアル（PDF）/ 不具合レポート（Excel）/ 製品仕様書（Word） |
| B — 金融 | 決算短信（PDF）/ ニュースソース（Web URL一覧）/ コンプライアンス規程（Word） |
| C — リテール/EC | 商品情報（PDF）/ FAQ（Excel）/ 注文データ（JSON） |
