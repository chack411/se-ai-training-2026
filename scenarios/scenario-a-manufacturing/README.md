## 3. 背景

東洋精密工業の品質管理部門は、技術者30名（ベテラン12名、若手18名）の体制で、2つの工場を横断して品質管理を担っている。月次約50〜80件の不具合に対応しており、不具合発生時の原因分析には平均2.5日を要する。その間のライン停止コストは1日あたり約500万円にのぼり、経営層からの改善要求が強まっている。

長年蓄積された技術マニュアル・不具合レポート・製品仕様書は、SharePoint、共有ファイルサーバー、紙の報告書に散在しており、過去の類似事例を探すだけで半日かかることもある。ベテラン技術者への知識集中と若手への伝承が進まないことが、組織の中長期的なリスクとなっている。

## 4. 現状の課題

1. 不具合発生時の原因分析に **平均2.5日** を要し、ライン停止コストが膨大（**1日あたり約500万円**）
2. 技術マニュアル・不具合レポート・製品仕様書が **SharePoint、共有ファイルサーバー、紙の報告書に散在**
3. 過去の類似不具合の検索は手作業（Excel のフィルタ、キーワード検索）で非効率
4. ベテラン技術者12名に知識が集中し、若手18名への **技術伝承が進んでいない**
5. IATF 16949 の監査対応に毎回 **2週間以上** の書類準備が必要

## 8. ヒアリング時の注意事項

- **一方的に提案しない** — まずは顧客の言葉で課題を引き出すことに徹する
- **「なぜ？」を繰り返す** — 表面的な要望の裏にある真のペインを探る
- **全員が発言する** — チーム全員が役割を持ち、多角的に質問する
- **メモを取る** — 後の設計・提案に向けて、具体的な数値・発言を記録する
- **時間配分を意識** — 全ポイントをカバーできるよう計画的に進行する
