## 3. 背景

ネクストマート株式会社は2012年設立の総合ECプラットフォームで、家電・ファッション・食品を中心に約3万SKUを取り扱っている。急成長に伴い、CS部門は東京・大阪・福岡の3拠点で120名体制にまで拡大したが、日次約3,500件の問い合わせに対し、オペレーターの情報検索に対応時間の約40%が消費されている。

顧客満足度（CSAT）は3.1 / 5と低迷しており、特に年4回のセール期間には問い合わせが最大3倍に急増し、対応が追いつかない状況が続いている。商品データベース、CRM、注文管理システムが別々に運用されており、オペレーターが案件ごとに複数システムを参照しなければならないのが根本原因となっている。

## 4. 現状の課題

1. 日次約 **3,500件** の問い合わせに対し、情報検索に対応時間の **約40%** を消費
2. 平均対応時間・待ち時間が目標を大幅に超過し、顧客満足度（CSAT）が **3.1 / 5** と低迷
3. セール期間の問い合わせ急増（**最大3倍**）への対応が困難
4. 商品 DB・CRM・注文管理システムが **分離** しており、オペレーターが複数システムを参照
5. セルフサービスチャネル未導入、多言語対応の不足

## 8. ヒアリング時の注意事項

- **一方的に提案しない** — まずは顧客の言葉で課題を引き出すことに徹する
- **「なぜ？」を繰り返す** — 表面的な要望の裏にある真のペインを探る
- **全員が発言する** — チーム全員が役割を持ち、多角的に質問する
- **メモを取る** — 後の設計・提案に向けて、具体的な数値・発言を記録する
- **時間配分を意識** — 全ポイントをカバーできるよう計画的に進行する
